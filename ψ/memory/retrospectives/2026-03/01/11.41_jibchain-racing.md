## Honest Feedback
1. **Environment Friction**: Not having Python or Node locally slowed down the initial "see the data" phase. I had to spend time writing a PowerShell HTTP server twice.
2. **Context Switching**: Jumping between Web3 (Viem), IoT (MQTT), and CV (Tailwind) required heavy context loading. I almost missed the scaling math for the Water Depth (10000x) in the first iteration.
3. **Link Persistence**: The CORS issue with `file://` protocols continues to be a recurring point of confusion for users. I should have suggested the GitHub Pages deploy sooner to avoid the "it's not loading" loop.

## Lessons Learned
- **Viem.js Precision**: When working with L1 contracts (like JIBCHAIN), unit formatting is everything. `int256` arrays need careful handling before passing to Chart.js.
- **MQTT Retain Power**: Retain flags are a legitimate and powerful substitute for small-scale databases in real-time apps.
- **PowerShell Versatility**: Net.HttpListener in PowerShell is a reliable dev server when traditional runtimes are missing.
